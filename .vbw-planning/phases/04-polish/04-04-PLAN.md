<objective>
Add the ONE E2E happy-path test that exercises the full install-generator output through the live Combustion harness:

1. Invoke `track_relay:install` programmatically (via `Rails::Generators.invoke`) targeting a tmpdir.
2. Copy/load the generator's output into the live `test/internal/` Combustion app at runtime.
3. Reload the catalog so the Railtie picks up `event :hello_world`.
4. Hit a real controller action that calls `track :hello_world, message: "..."`.
5. Assert the Test subscriber captured the event with the correct typed payload via `assert_tracked`.
6. Teardown: delete the loaded artifacts so subsequent test runs are clean.

This proves the install generator's "works out of the box" claim: the files it produces actually load, the catalog actually registers, and a tracked controller call actually fans out to subscribers.

**Open-question resolution 3 (E2E approach):** Programmatic `Rails::Generators.invoke` (NOT write-inline). Rationale: invoking the actual generator binary is a stronger signal than mimicking its output by hand; the structural tests (plan 04-03) already prove file content shape, so this test adds value precisely by exercising the runner pipeline. The boot-ordering risk research called out is sidestepped because we invoke into a tmpdir first, then `cp`/`load` files into `test/internal/` at test runtime AFTER Combustion is fully booted.

Depends on plan 04-01 (install generator must exist) and plan 04-03 (E2E reuses the test-isolation patterns and confirms structural tests pass first; running E2E with broken structural assertions is a wasted signal).
</objective>

Create the E2E test:

```ruby
# frozen_string_literal: true

require "test_helper"
require "rails/generators"
require "fileutils"
require "generators/track_relay/install/install_generator"
require "track_relay/testing/helpers"

class GeneratorInstallE2ETest < ActionDispatch::IntegrationTest
  include TrackRelay::Testing::Helpers

  TMPDIR = File.expand_path("../../tmp/generator_e2e", __dir__)
  INTERNAL_ROOT = File.expand_path("../internal", __dir__)
  CATALOG_DIR = File.join(INTERNAL_ROOT, "config/track_relay")
  SUBSCRIBERS_DIR = File.join(INTERNAL_ROOT, "app/track_relay/subscribers")
  GENERATED_FILES = [
    File.join(CATALOG_DIR, "sample.rb"),
    File.join(SUBSCRIBERS_DIR, "application_subscriber.rb")
  ].freeze

  setup do
    # 1. Run the install generator into a clean tmpdir.
    FileUtils.rm_rf(TMPDIR)
    FileUtils.mkdir_p(TMPDIR)
    Rails::Generators.invoke(
      "track_relay:install",
      [],
      destination_root: TMPDIR,
      shell: Thor::Shell::Basic.new # silence color output in CI
    )

    # 2. Copy ONLY the generator's catalog + subscriber output into test/internal.
    #    Do NOT copy the generated initializer (test/internal manages its own
    #    config; the initializer would force load order issues against the
    #    already-booted Combustion app). Do NOT copy the inject_into_class
    #    output for ApplicationController — test/internal already has the
    #    include, and the generator's idempotency guard skipped it anyway.
    FileUtils.mkdir_p(CATALOG_DIR)
    FileUtils.cp(
      File.join(TMPDIR, "config/track_relay/sample.rb"),
      File.join(CATALOG_DIR, "sample.rb")
    )
    FileUtils.mkdir_p(SUBSCRIBERS_DIR)
    FileUtils.cp(
      File.join(TMPDIR, "app/track_relay/subscribers/application_subscriber.rb"),
      File.join(SUBSCRIBERS_DIR, "application_subscriber.rb")
    )

    # 3. Force the Railtie to_prepare-style catalog reload.
    TrackRelay::Catalog.clear!
    Dir.glob(File.join(CATALOG_DIR, "**/*.rb")).sort.each { |f| load f }

    # 4. test_mode! activates the Test subscriber for assert_tracked.
    TrackRelay.test_mode!

    # 5. Restart the dispatcher so events fan out to subscribers.
    #    test_helper.rb's global teardown called Dispatcher.stop! after the
    #    previous test, and TrackRelay.test_mode! does NOT (re)start dispatch
    #    — it only swaps the subscriber list. start! is idempotent.
    TrackRelay::Dispatcher.start!
  end

  teardown do
    GENERATED_FILES.each { |f| File.delete(f) if File.exist?(f) }
    # Tear down empty dirs in reverse order. rescue nil because rmdir
    # raises if a dir is non-empty (defensive — should be empty after deletes).
    [CATALOG_DIR, SUBSCRIBERS_DIR, File.dirname(SUBSCRIBERS_DIR)].each do |dir|
      Dir.rmdir(dir) if Dir.exist?(dir) && Dir.empty?(dir)
    rescue Errno::ENOENT, Errno::ENOTEMPTY
      # ignore
    end
    FileUtils.rm_rf(TMPDIR)
    TrackRelay.test_mode_off!
    # ActiveSupport::TestCase global teardown (test_helper.rb:43-49) handles
    # Catalog.clear!, Dispatcher.stop!, and reset_config! — no-op here.
  end

  test "install generator output: tracked controller call captured by Test subscriber" do
    get hello_path(message: "hi from e2e")
    assert_response :ok
    assert_tracked :hello_world, message: "hi from e2e"
  end
end
```

Notes:
- `Rails::Generators.invoke` is the documented programmatic entry point (Rails Generators public API). It runs the generator end-to-end including all action methods.
- The test deliberately copies ONLY the catalog + subscriber files — the initializer is excluded because (a) test/internal manages its own config via `test_helper.rb` setup hooks, and (b) re-loading an initializer mid-test risks subscriber double-registration that the global teardown can't fully clean up.
- The inject_into_class output for ApplicationController is also NOT copied — test/internal already has the include, and the generator's idempotency guard from plan 04-01 means the generator's output for that file is identical to the existing one. Copying would be a no-op, but verifying we DON'T need to is the cleaner statement.
- `test_helper.rb` requires `track_relay/testing` (not `testing/helpers`), so this file requires the helpers explicitly: `require "track_relay/testing/helpers"`. Without that require, `TrackRelay::Testing::Helpers` is undefined when the `include` runs.
- `test_helper.rb`'s global teardown calls `Dispatcher.stop!` after every test, so each integration test that depends on dispatch fan-out must call `Dispatcher.start!` in its own setup. `TrackRelay.test_mode!` only swaps the subscriber list — it does not (re)start the dispatcher. The teardown does NOT need a corresponding `Dispatcher.stop!` here — global teardown already handles it.
- `Dir.empty?` exists since Ruby 2.4. Compatible with the gem's Ruby 3.2 floor.
- Teardown uses defensive `rescue Errno::ENOENT, Errno::ENOTEMPTY` — if a future test adds files to those dirs, we don't want this teardown to spuriously fail.
