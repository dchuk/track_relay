<objective>
Ship `rails g track_relay:event NAME` and `rails g track_relay:subscriber NAME` — both `Rails::Generators::NamedBase` generators that produce opinionated working stubs.

Event generator creates a single new file `config/track_relay/<file_name>.rb` with a `TrackRelay.catalog do ... end` block defining the named event. The Railtie's `Dir.glob("config/track_relay/**/*.rb")` will pick it up at boot/reload.

Subscriber generator creates a single new file `app/track_relay/subscribers/<file_name>_subscriber.rb` (path matches the install generator's `ApplicationSubscriber` location — convention locked in plan 04-01).

Depends on plan 04-01 because the subscriber path convention (`app/track_relay/subscribers/`) is established there. Running this plan after 04-01 keeps the path consistent — if 04-01 changes the path during implementation, this plan inherits it without rework.
</objective>

Create the event generator. Required structure:

```ruby
# frozen_string_literal: true

require "rails/generators"

module TrackRelay
  module Generators
    class EventGenerator < Rails::Generators::NamedBase
      source_root File.expand_path("templates", __dir__)

      desc "Creates a typed catalog entry stub at config/track_relay/<name>.rb."

      def create_event_file
        template "event.rb.tt", "config/track_relay/#{file_name}.rb"
      end
    end
  end
end
```

Notes:
- `Rails::Generators::NamedBase` (NOT Base) since this generator takes a NAME argument. NamedBase parses `NAME` from `argv[0]` and exposes `file_name` and `class_name`.
- Single action method `create_event_file` — produces ONE file per generator invocation. Multiple events = multiple invocations = multiple files. The Railtie's catalog loader merges them at boot.
- Destination uses NamedBase's `file_name` (snake_case): `rails g track_relay:event ArticleViewed` produces `config/track_relay/article_viewed.rb`; `rails g track_relay:event article_viewed` produces the same.
- Single sentence `desc` matching Rails convention.
