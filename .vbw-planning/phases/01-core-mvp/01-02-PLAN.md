Write `lib/track_relay/event_payload.rb`:
- `class EventPayload` with `attr_reader :definition, :params, :context, :timestamp`.
- Constructor: `initialize(definition:, params:, context: {}, timestamp: Time.now)`. Stores raw input ÔÇö does NOT validate yet.
- `name` returns `definition.name`.
- `validate!` ÔÇö validates `params` against `definition.params`:
  1. For each schema: if `sanitize` is callable, run it on the raw value first.
  2. If value is missing AND schema.required, raise `ValidationError` with key name.
  3. If value present, coerce per schema.type:
     - `:integer` Ôćĺ `Integer(v)` (raises on bad input, surfaced as ValidationError).
     - `:string` Ôćĺ `String(v)`.
     - `:float` Ôćĺ `Float(v)`.
     - `:boolean` Ôćĺ strict: `true/false/'true'/'false'/1/0` Ôćĺ bool; else raise.
     - `:datetime` Ôćĺ `Time.iso8601(v)` if String, accept Time/DateTime as-is, else raise.
  4. Apply `max` (raise ÔÇö never silently truncate).
  5. Apply `in` (inclusion list).
  6. Apply `format` (regex match).
  7. Reject params not declared in schema unless `definition` is `nil` (untyped path ÔÇö see Plan 04 ÔÇö but for typed validate, extras raise).
  Returns coerced params Hash. Mutates `@params` to coerced form.
- `to_h` ÔÇö returns `{name: definition.name, params: coerced_params, context: context, timestamp: timestamp.iso8601}` for serialization (DeliveryJob in Plan 05 will use this).
- For untyped events (definition == nil), `validate!` is a no-op and `to_h` includes `name` from a separately stored `@name` attribute. Implement constructor variant `EventPayload.untyped(name:, params:, ...)`.

Tests in `test/unit/event_payload_test.rb`:
- Required missing param raises ValidationError naming the key.
- Integer coercion: `"42"` Ôćĺ `42`. Bad input raises.
- String coercion + max-length raise (NOT silent truncation).
- Sanitize callable runs before validation: `sanitize: ->(v) { v.truncate(100) }` lets a 200-char input pass when paired with `max: 100`.
- `in:` mismatch raises.
- `format:` mismatch raises.
- Boolean strict-mode coerces "true"/"false"/1/0 correctly; other strings raise.
- Extra param (not in schema) raises ValidationError.
- Untyped variant skips validation.
- `to_h` returns serializable hash with iso8601 timestamp.

Use TDD-cycle skill rigorously here ÔÇö this is the central runtime class.
