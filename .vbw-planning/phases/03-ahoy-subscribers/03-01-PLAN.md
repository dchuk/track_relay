## Goal

Ship the server-side `TrackRelay::Subscribers::Ahoy` subscriber: a synchronous, duck-typed wrapper that routes catalog events through `Current.controller.ahoy.track` (Ahoy's only public tracking surface) and skip-logs when no controller is in scope (job/console context). Add `ahoy_matey` as a development dependency so unit/integration tests can import the gem when they need to assert subscriber behavior. Lays the foundation for Phase 03's REQ-09 server half.

## Resolved Deviations

- **MH-06 (formerly self-recorded as DEVN-02):** ROADMAP REQ-09 originally specified routing via `TrackRelay::Current.controller.ahoy.track or visit.track only`. The phrase `visit.track` references an API that does not exist on `Ahoy::Visit` ‚Äî `Ahoy::Tracker` is Ahoy's only public tracking surface (see `03-RESEARCH.md ¬ß2`). The implemented contract:
  - **Controller-present path:** dispatch via `controller.ahoy.track(payload.name.to_s, payload.params)`.
  - **No-controller path (job/console):** skip-log via `Rails.logger.warn` and return ‚Äî this is the substitute for the missing `visit.track` fallback.
  Both paths are pinned by unit tests `test_dispatches_via_controller_ahoy_track` and `test_skips_when_no_current_controller` in `test/unit/subscribers/ahoy_test.rb`. Original self-deviation captured in `03-01-SUMMARY.md` deviations[] as DEVN-02.

## Notes

**Why synchronous, in one line:** `Ahoy::Tracker` is bound to the live request (it wraps the controller cookie jar and visit lifecycle), so the only safe place to call `tracker.track` is on the request thread before `Rails.application.executor.wrap` clears `CurrentAttributes`. GA4's async path is appropriate because GA4 is an external HTTP call; Ahoy's `tracker.track` is an in-process DB write ‚Äî synchronous is idiomatic AND structurally required.

**Why duck-typing instead of `require "ahoy"`:** The gem must load cleanly in non-Ahoy host apps (REQ contract). `controller.respond_to?(:ahoy, true)` returns false when `Ahoy::Controller` was never `include`d, and the subscriber gracefully skips. Precedent: `lib/track_relay/client_id/ahoy_visitor.rb` uses the exact same pattern with no top-level `require "ahoy"`.

**Why stubbed tracker in tests:** Real `Ahoy::Tracker` calls `BaseStore#exclude?` which checks `request.user_agent` for bots and `Rails::HealthController`. In the test harness with no real request, this can return true (skip) or raise (NoMethodError on nil request). Stubbing the tracker isolates the subscriber's contract from Ahoy-internal flakiness.

**`visit.track` deviation:** ROADMAP's success criteria mention `visit.track` as a fallback dispatch path. The research confirms `Ahoy::Visit` is an ActiveRecord model with no `track(name, props)` method ‚Äî `Ahoy::Tracker` is Ahoy's only public tracking surface. The implementation routes through `controller.ahoy.track` only and treats no-controller as the skip case. This deviation is captured in `must_haves` so QA gate sees it; it does NOT silently subset REQ-09 ‚Äî it documents that the original criterion was based on an API that does not exist.
