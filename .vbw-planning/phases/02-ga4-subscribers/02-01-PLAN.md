## Goal

Add the foundations every later GA4 plan depends on: subscriber-side `only:`/`except:` event filters wired into `Subscribers::Base#handle`, a public `TrackRelay.subscribe(klass, only:, except:)` registration helper, and `webmock` as a dev dependency so plan 02-04 can stub the GA4 HTTP endpoint without hitting the network.

## Files Touched

- `track_relay.gemspec` — add `spec.add_development_dependency "webmock", "~> 3.23"` (currently dev deps end at line 30-40 per Scout §1)
- `test/test_helper.rb` — add `require "webmock/minitest"` and `WebMock.disable_net_connect!(allow_localhost: true)` (Scout §9)
- `lib/track_relay/subscribers/base.rb` — extend with `class_attribute :only_events, :except_events` (default nil), a `filter(only: nil, except: nil)` class DSL method, a private `#filtered?(event_name)` instance helper, and short-circuit at the top of `#handle` BEFORE the sync/async branch (Scout §1 base.rb:56-63, Scout §4 sketch lines 305-330)
- `lib/track_relay.rb` — add public module method `TrackRelay.subscribe(subscriber_or_class, only: nil, except: nil)` that:
  - accepts either a class (instantiates it) or an instance,
  - sets `only_events`/`except_events` on the instance (instance-level override of class defaults),
  - delegates to `config.subscribe(instance)`
- `test/unit/subscribers/base_filter_test.rb` (new) — unit tests for filter behavior
- `test/unit/track_relay_subscribe_test.rb` (new) — tests for the `TrackRelay.subscribe` helper

## Tasks

1. **Add webmock dev dependency + wire test_helper.** Add `spec.add_development_dependency "webmock", "~> 3.23"` to `track_relay.gemspec`. Add `require "webmock/minitest"` and `WebMock.disable_net_connect!(allow_localhost: true)` near the top of `test/test_helper.rb` after the existing requires. Run `bundle install` and `bin/rails test` (or `bundle exec rake`) to confirm the suite still passes. Acceptance: `bundle exec ruby -e 'require "webmock"; puts WebMock::VERSION'` succeeds.

2. **Write filter tests (RED).** Per `tdd-cycle`, add `test/unit/subscribers/base_filter_test.rb` with cases: (a) subclass with `filter only: %i[purchase]` only receives `purchase`, drops `sign_up`; (b) subclass with `filter except: %i[page_view]` drops `page_view` but receives others; (c) no filter set = receives every event; (d) filter check runs BEFORE `safe_deliver` — assert that a subscriber whose `deliver` raises does NOT log when the event is filtered out (use `assert_no_match` against `Rails.logger` capture). Use a tiny in-memory test subscriber (synchronous, captures into an array) to drive each case. Run tests; confirm RED.

3. **Implement filter logic in Base (GREEN).** In `lib/track_relay/subscribers/base.rb` add `class_attribute :only_events, :except_events` defaulting to nil (use `Set` semantics — store as `Set<Symbol>` after the DSL coerces). Add class method `filter(only: nil, except: nil)` that assigns `Set.new(Array(only).map(&:to_sym))` (or nil) to each. Add private `#filtered?(name)` returning `false` if both sets nil, `!only_events.include?(name)` if `only` set, else `except_events.include?(name)`. At the top of `#handle` (currently base.rb:56-63), `return nil if filtered?(payload.name.to_sym)` BEFORE the sync/async branch. Run filter tests; confirm GREEN. Run full Phase 1 suite; confirm no regressions.

4. **Add `TrackRelay.subscribe` helper with tests (RED → GREEN).** Write `test/unit/track_relay_subscribe_test.rb` covering: (a) `TrackRelay.subscribe(MySubscriber)` instantiates and registers; (b) `TrackRelay.subscribe(MySubscriber, only: %i[purchase])` sets `only_events` on the INSTANCE (not the class — overrides class defaults without mutating other instances); (c) `TrackRelay.subscribe(MySubscriber.new)` accepts a pre-built instance. Then implement `TrackRelay.subscribe` in `lib/track_relay.rb` (near the existing `track`, `identify`, `catalog`, `configure` methods); use `singleton_class.attr_accessor :only_events, :except_events` per-instance OR `instance_variable_set` + per-instance reader override. Confirm tests pass and Phase 1 suite still green.

5. **Refactor + standardrb pass.** Run `bundle exec standardrb --fix` over the touched files. Confirm `bundle exec rake` is green. Update CHANGELOG.md `[Unreleased]` section with bullets: "Subscriber `only:`/`except:` filters via `filter` class DSL or `TrackRelay.subscribe(klass, only:, except:)`", "webmock added as dev dependency for HTTP-stubbed subscriber tests".
