## Goal

Generate a typed JSON manifest of the catalog at `public/track_relay_catalog.json` that the JS client (Plan 02-05) can fetch to validate events client-side. Wire it into `assets:precompile` for production builds and `config.to_prepare` for dev-mode regeneration. Implements REQ-12 + REQ-28.

## Files Touched

- `lib/track_relay/manifest.rb` (new) — `TrackRelay::Manifest` module with `.generate(catalog:)` returning the manifest Hash and `.write!(path:)` writing pretty JSON to disk; references `TrackRelay::Catalog.all` (Scout §6 confirms `Catalog.all` returns frozen Array, `EventDefinition#params` is `Hash{Symbol => ParamSchema}` with `type` attribute, and types are `:integer/:string/:float/:boolean/:datetime` per Scout §6)
- `lib/tasks/track_relay.rake` — add the `track_relay:manifest` namespace task. Existing rake file has `track_relay:lint` and `track_relay:lint:json` tasks (Phase 1, per Scout §1 + repo listing); new task lives alongside them
- `lib/track_relay/railtie.rb` — add two new initializer blocks:
  - `track_relay.enhance_assets_precompile` (Scout §6 line 494-498): on initialization, if `defined?(Rake) && Rake::Task.task_defined?("assets:precompile")`, call `Rake::Task["assets:precompile"].enhance(["track_relay:manifest"])`. The `defined?(Rake) &&` prefix is REQUIRED: Combustion boots the gem's dummy app (`test/internal/`) with only `:action_controller, :active_job` — neither of those frameworks requires `rake` at app boot — so a bare `Rake::Task` reference would raise `NameError` during the test suite's app-initialization phase. The `defined?(Rake) &&` guard mirrors the defensive posture of `task_defined?` itself (which already tolerates the task being undefined) and matches the shipped code at `lib/track_relay/railtie.rb:85-87`.
  - Modify the existing `track_relay.catalog_autoload` initializer (currently at `lib/track_relay/railtie.rb:45-49`): inside the `config.to_prepare` block, AFTER the existing catalog reload, when `Rails.env.development?` and `Catalog.all.any?`, call `TrackRelay::Manifest.write!`
- `test/unit/manifest_test.rb` (new) — unit tests for `Manifest.generate` and `Manifest.write!`
- `test/integration/manifest_rake_test.rb` (new) — drives the `track_relay:manifest` rake task end-to-end against the dummy app's catalog
- `test/integration/manifest_dev_reload_test.rb` (new) — exercises the `to_prepare` branch by toggling env to development and invoking the reload callback (or by stubbing `Rails.env.development?`)
- `lib/track_relay/manifest.rb` is required by `lib/tasks/track_relay.rake` via `require_relative "../track_relay/manifest"` to avoid touching `lib/track_relay.rb` (keeps file-disjoint with Plan 02-02)

## Tasks

1. **Write `Manifest.generate` tests + implement (RED → GREEN).** Per `tdd-cycle`: in `test/unit/manifest_test.rb`, build a small catalog using the existing DSL (Scout §1 confirms catalog DSL access via `TrackRelay.catalog { event :purchase do float :value, required: true; string :currency end }`). Assert `TrackRelay::Manifest.generate` returns the exact 02-CONTEXT shape: `{version: TrackRelay::VERSION, generated_at: ISO8601-string, events: {"purchase" => {params: {"value" => "float", "currency" => "string"}, required: ["value"]}}}`. Cover all 5 types (`integer/string/float/boolean/datetime`) and a no-required-params event (the `required` key should still exist but be `[]`). Implement `lib/track_relay/manifest.rb` per Scout §6 (lines 446-466). Confirm GREEN.

2. **Write `Manifest.write!` tests + implement.** Test: `Manifest.write!(path: tmpfile)` writes pretty-printed JSON, returns path, parseable. Add `test_creates_parent_directory`: pass `path: Pathname(Dir.mktmpdir).join("brand_new_subdir/track_relay_catalog.json")`, assert `File.exist?(path)` is true after the call AND `File.directory?(File.dirname(path))` is true (the missing `brand_new_subdir/` was created). This is REQUIRED because the Combustion dummy app at `test/internal/` ships without a `public/` directory — without `mkdir_p`, the very first `Manifest.write!` call in the gem's own test suite crashes with `Errno::ENOENT`. Implement using `require "fileutils"; FileUtils.mkdir_p(File.dirname(path)); File.write(path, JSON.pretty_generate(generate))`. Default path = `Rails.root.join("public", "track_relay_catalog.json")` — but in unit tests use a `Tempfile`/`Dir.mktmpdir`. Confirm tests pass.

3. **Add `rake track_relay:manifest` task (RED → GREEN).** Add `test/integration/manifest_rake_test.rb`. Cases: (a) with a populated catalog, `Rake::Task["track_relay:manifest"].invoke` writes the JSON to `Rails.root.join("public", "track_relay_catalog.json")` and the file content matches `Manifest.generate`; (b) with `Catalog.clear!` first, the task aborts with non-zero exit and a clear message ("aborting: catalog is empty"). Implement the task in `lib/tasks/track_relay.rake`: `desc "Generate public/track_relay_catalog.json from the loaded catalog"; task manifest: :environment do; abort("[track_relay] aborting: catalog is empty") if TrackRelay::Catalog.all.empty?; path = TrackRelay::Manifest.write!; puts "[track_relay] manifest written to #{path} (#{TrackRelay::Catalog.all.size} events)"; end`. Use `require_relative` to load the Manifest module. Note: invoke + reset Rake task between tests (`Rake::Task["track_relay:manifest"].reenable`).

4. **Wire Railtie hooks (RED → GREEN).** Add `test/integration/manifest_dev_reload_test.rb`. Cases: (a) when `Rails.env.development?` is true and the `to_prepare` callback fires (call it via `Rails.application.reloader.prepare!` or by directly invoking the inner block), the manifest file is rewritten (compare mtime or content); (b) when `Rails.env.test?` (default), the `to_prepare` block does NOT write the manifest (avoid every-test churn). For `assets:precompile`: since invoking the real `assets:precompile` is heavy, test-strategy is to assert the prerequisite is present — `Rake::Task["assets:precompile"].prerequisites.include?("track_relay:manifest")` (after the gem boots in a test that defines `assets:precompile` with `Rake::Task.define_task("assets:precompile")`). Implement: in `lib/track_relay/railtie.rb`, add `initializer "track_relay.enhance_assets_precompile" do; if defined?(Rake) && Rake::Task.task_defined?("assets:precompile"); Rake::Task["assets:precompile"].enhance(["track_relay:manifest"]); end; end`. The `defined?(Rake) &&` short-circuit is non-optional — Combustion boots the dummy app with `:action_controller, :active_job` only, so `Rake` is NOT loaded at app-init time inside the gem's own test suite; without the guard, the initializer raises `NameError: uninitialized constant Rake` on every `bundle exec rake` run. The guard is symmetric with `task_defined?`'s own defensive posture (it tolerates an undefined task name) and matches the AP-03–verified shipped implementation at `lib/track_relay/railtie.rb:85-87`. Modify the `track_relay.catalog_autoload` block to append `TrackRelay::Manifest.write! if Rails.env.development? && TrackRelay::Catalog.all.any?` after the catalog reload. Confirm GREEN.

5. **Refactor + CHANGELOG.** `bundle exec standardrb --fix`. Update CHANGELOG.md `[Unreleased]` with: "JSON manifest generation: `rake track_relay:manifest` writes typed `public/track_relay_catalog.json`; auto-runs before `assets:precompile`; regenerates on `to_prepare` in development."
