## Goal

Replace Phase 1's hardcoded `_ga`-cookie-only `client_id` resolution with a configurable, ordered resolver chain (`Ga` → `AhoyVisitor` → `Session`) that runs once per request inside the existing `ControllerTracking` `before_action` and memoizes the result on `TrackRelay::Current.client_id`. Implements REQ-26 from REQUIREMENTS.md.

- `lib/track_relay/client_id/ahoy_visitor.rb` (new) — Scout §5 sketch at lines 406-416

- `lib/track_relay/controller_tracking.rb` — replace `_track_relay_client_id_from_cookie` (currently lines 59,62-72) with a private `_resolve_client_id` that iterates `TrackRelay.config.client_id_resolvers`, rescues StandardError per resolver, returns first non-nil. Update `_track_relay_set_current` (lines 56-60) to assign `Current.client_id = _resolve_client_id`

4. **Rewire `ControllerTracking` to use the chain (RED → GREEN).** Add `test/integration/client_id_chain_test.rb` (a dummy-app controller test). Cases: (a) baseline — `_ga` cookie present → `Current.client_id` matches `ClientId::Ga` output (parity with Phase 1); (b) no `_ga` + AhoyVisitor configured returning a token → `Current.client_id` is the token; (c) all three resolvers return nil except a custom resolver inserted at position 0 → that one wins; (d) one resolver raises StandardError → chain continues and returns the next resolver's value; (e) UUID-fallback path is stable across two before_action invocations within the same session. Confirm RED. Then in `lib/track_relay/controller_tracking.rb`: remove `_track_relay_client_id_from_cookie` (lines 62-72), add `_resolve_client_id` private method that iterates `TrackRelay.config.client_id_resolvers`, wraps each `resolver.call(controller: self)` in `rescue StandardError`, returns first non-nil. Update `_track_relay_set_current` (lines 56-60) to call it. Confirm GREEN.
